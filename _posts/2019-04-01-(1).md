## 데이터베이스 개념

**데이터베이스의 정의**<br>
    : 특정 조직의 업무를 수행하는데 필요한 상호 관련된 데이터들의 모임<br>

데이터의 4가지 종류

    통합데이터 (Integrated Data) : 중복이 최소화된 Data
    저장데이터 (Stored Data) : 컴퓨터가 접근 가능한 저장매체에 저장된 Data
    운영데이터 (Operational Data) : 조직의 목적을 위해 존재가치가 확실한 Data
    공유데이터 (Shared Data) : 여러 응용 프로그램들이 공동으로 사용하는 Data

**DB의 특징**<br>

    실시간 접근성 (Real Time Accessibility) : 사용자의 질의에 대하여 즉시처리 응답하는 특징
    계속적인 진화 (Continuous Evolution) : 삽입, 삭제, 갱신을 통해 항상 최신 데이터를 유지
    동시 공유 (Concurrent Sharing) : 여러 사용자가 동시에 원하는 데이터를 공용으로 사용 할 수 있는 특징
    내용에 의한 참조 (Content Reference) : DB에 있는 데이터를 참조할때 사용자가 요구하는 데이터 내용에 따라 참조
    중복의 최소화 (Redundary Minimize) : 중복성을 최소화

**DDL (Data Define Language) 데이터 정의어**<br>
    : DB구조, 데이터 형식 등을 구축하거나 변경할 목적으로 사용 (CREATE, ALTER, DROP)

**DML (Data Manipulation Language) 데이터 조작어**<br>
    : 응용프로그램과 데이터베이스 관리시스템간의 인터페이스를 위한 언어 (SELETE, INSERT, UPDATE, DELETE)

**DCL (Data Control Language) 데이터 제어어**<br>
    : 보안 및 권한제어, 무결성, 회복, 병행제어를 위한 언어 (GRANT, REVOKE, COMMIT, ROLLBACK)

**데이터베이스관리자 (DBA; Data Base Administrator)**<br>
    : DDL과 DCL을 통해 데이터베이스를 정의하고 제어하는 사람

**데이터관리자 (DA; Data Administrator)**<br>
    : 하나의 기업 또는 조직 내에서 데이터에 대한 정의, 체계화, 감독 및 보안, 담당을 하거나 관리를 총괄한다.

**데이터설계자 (DA; Data Architect)**<br>
    : 데이터의 구조를 체계적으로 정의하는 사람

**데이터베이스 관리시스템 (Database Management System)**<br>
    : 사용자와 데이터베이스 사이에서 사용자의 요구에 따라 정보를 생성해주고 데이터베이스를 관리해주는 시스템

**DBMS의 장·단점**<br>

    1. 데이터의 논리적, 물리적 독립성이 보장된다.
    2. 데이터를 공동으로 사용 가능하다.
    3. 데이터의 무결성을 유지 할 수 있다.
    4. 데이터의 실시간 처리가 가능하다.
    5. 데이터의 중복을 피할 수 있다.
    6. 데이터 통합 관리, 일관성 유지, 보안유지, 최신화 유지

**데이터 스트림 관리시스템 (DSMS; Data Stream Management System)**<br>
    : 무선 센서 네트워크나 인터넷 같은 통신 상에서 발생하는 대량의 스트림 데이터를 처리하고 관리하는 시스템

**스키마(Schema)의 개념**<br>
    : DB를 구성하는 데이터 개체, 속성, 관계 및 데이터 값들이 갖는 제약 조건 등에 관해 전반적으로 정의한다.

**데이터 사전 (Data Dictionary)**<br>
    : 데이터베이스에 저장되어 있는 모든 데이터 개체들에 대한 정보를 유지·관리하는 시스템, **`시스템카탈로그`**라고도 불린다. 또는 메타데이터

**메타데이터 (Meta Data)**<br>
    : 실제 저장되는 데이터는 아니지만, 저장되는 데이터와 직접 혹은 간접적으로 관계가 있는 정보를 제공하는 데이터를 말한다.

**외부 스키마**<br>
    : 사용자나 응용프로그래머나 각 개인의 입장에서 필요로 하는 데이터베이스의 논리적 구조를 정의

**개념 스키마**<br>
    : 개체 간의 관계나 제약조건을 나타내고 정책 및 규정에 관한 명세를 정의

**내부 스키마**<br>
    : 데이터베이스의 물리적 구조를 정의

관계형 데이터구조 : 릴레이션<br>
관계형 데이터 모델에서는 데이터를 원자값으로 갖는 이차원의 테이블로 표현, 이것이 **릴레이션**<br>
릴레이션의 구조를 나타내는 것이 **릴레이션스키마**<br>
릴레이션의 실제 값들이 **릴레이션인스턴스**<br>
릴레이션의 Degree는 (속성의 수 = 차수)<br>
릴레이션의 Cardinality (튜플의 수 = 기수 = 레코드의 수)<br>

개념적 데이터 모델인 **E-R모델**을 논리적데이터 모델인 **릴레이션 스키마**로 변환 하는 것이, **매핑룰(Mapping Rule)**이다.<br>
**속성**은 **컬럼(Column)**으로, **식별자**는 **기본키**로, 릴레이션 간의 관계는 **기본키**와 이를 참조하는 **외래키**를 이용하여 표현<br>

식별관계 (Identifying)<br>
    : 개체 A, B 사이의 관계에서 A개체의 기본키가 B개체의 외래키이면서 동시에 기본키가 되는 관계를 말한다.
  
비식별관계 (Non-identifying)<br>
    : 개체 A, B사이의 관계에서 A개체의 기본키가 B개체의 비기본키 영역에서 외래키가 되는 관계를 말한다.

### 키의 종류

![](/static/assets/img/blog/2019-04-01-(1)/2019-04-02-18-54-31.png){: .wh35 .center}

![](/static/assets/img/blog/2019-04-01-(1)/2019-04-02-18-55-12.png){: .wh35 .center}


슈퍼키 (Super key)<br>
    : 모든 튜플에 대해서 유일성은 만족하지만, 최소성은 만족하지 못한다.

후보키 (Candidate key)<br>
    : 속성들 중에서 튜플을 유일하게 식별하기위해 사용되는 속성들의 부분집합.

기본키 (Primary key)<br>
    : 후보키 중에서 특별히 선정된 키로 중복된 값을 가질 수 없다.<br>
  유일성과 최소성을 가지며 튜플을 식별하기 위해 반드시 필요한 키이다.

대체키 (Alternate key)<br>
    : 후보키 중에서 선정된 기본키를 제외한 나머지 후보키를 말한다.

외래키 (Foreign key)<br>
    : 다른 릴레이션의 기본키를 참조하는 속성 or 속성들의 집합.

유일성이란?<br>
    : 하나의 키 값으로 하나의 튜플만을 유일하게 식별 할 수 있어야한다.

최소성이란?<br>
    : 키를 구성하는 속성 하나를 제거하면 유일하게 식별할 수 없도록 꼭 필요한 최소의 속성으로 구성되어야 한다.

무결성 (Integrity)의 개념<br>
    : DB에 저장된 값과 그것이 표현하는 현실세계의 실제 값이 일치하는 정확성을 의미한다.

### 무결성의 종류

- NULL 무결성 : 특정 송성 값이 NULL이 될 수 없도록 하는 규정

- 고유 무결성 : 릴레이션의 특정 속성에 대해 각 튜플이 갖는 속성 값들이 서로 달라야 한다는 규정

- 도메인 무결성 : 특정 속성의 값이 그 속성이 정의된 도메인에 속한 값이어야한다는 규정

- 키 무결성 : 하나의 릴레이션에는 적어도 하나의 키가 존재해야한다는 규정

- 관계 무결성 : 릴레이션에 어느 한 튜플의 삽입 가능 여부 또는 한 릴레이션과 다른 릴레이션의 튜플사이의 관계에 대한 적절성 여부를 지정한 규정

- 참조 무결성 : 외래키 값은 NULL이거나 참조 릴레이션의 기본키 값과 동일해야 한다는 규정

- 개체 무결성 : 기본 릴레이션의 기본키를 구성하는 어떤 속성도 NULL 일 수 없다는 규정

이상(Anomaly)<br>
    : 테이블에서 속성들의 종속으로 인해 데이터의 중복이 발생하고 이 중복으로 인해 테이블 조작 시 문제가 발생하는 현상을 말한다.

삽입이상<br>
    : 테이블에 데이터를 삽입할 때 의도와는 상관없는 값들도 함께 삭제되는 연쇄 삭제가 발생하는 현상이다.

삭제이상<br>
    : 테이블에서 한 튜플을 삭제할때 의도와는 상관없는 값들도 함께 삭제되는 연쇄 삭제가 발생하는 현상이다.

갱신이상<br>
    : 튜플에 있는 속성 값을 갱신 할 때 일부 튜플의 정보만 갱신되어 정보에 불일치성이 생기는 현상

X -> Y 관계일때 X를 결정자, Y를 종속자라고 한다.


## 정규화

정규화(Normalization)의 개념<br>
    : 테이블을 무손실 분해하는 과정으로, 정규화의 목적은 가능한 중복을 제거하여 삽입, 삭제, 갱신 이상의 발생 가능성을 줄이는 것입니다.

- 제 1 정규형<br>
  - 릴레이션에 속한 모든 속성의 도메인이 원자값만으로 되어있는 정규형이다.

- 제 2 정규형<br>
  - 릴레이션이 제 1정규형이고, 기본키가 아닌 모든 속성이 기본키에 대하여 완전함수적 종속을 만족하는 정규형이다.
    즉, 부분적 함수적 종속이 제거된 형태

- 제 3 정규형<br>
  - 릴레이션이 제 2정규형이고, 기본키가 아닌 모든 속성이 기본키에 대해 이행적 함수적 종속을 만족하지 않는 정규형이다.

- BCNF<br>
  - 릴레이션에서 모든 결정자가 후보키인 정규형이다.

- 제 4 정규형<br>
  - 릴레이션에서 다중값종속 A ->> B가 존재할 경우 릴레이션의 모든 속성이 A에 함수적 종속 관계를 만족하는 경우

- 제 5 정규형<br>
  - 릴레이션에서 모든 조인 종속이 릴레이션의 후보키를 통해서만 성립되는 정규형이다.

트랜잭션(Transaction) : DB에서 하나의 논리적 기능을 수행하기 위한 일련의 연산집합으로서 작업의 단위이다.

트랜잭션의 특성

Atomicity(원자성) : DB에 모두 반영되든지 아니면 전혀 반영되지 않아야한다.

Consistency(일관성) : 일관성있게 트랜잭션 수행 전과 후의 상태가 같아야한다.
